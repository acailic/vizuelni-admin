Chart Types

Column & Bar Charts

Comparing values across categories

Use when

  • •Category rankings
  • •Comparisons across regions
  • •Deviation from target
  • •Fewer than 10 categories

Avoid when

  • •More than 15 categories
  • •Trends over time
  • •Proportions (use pie instead)
  • •Negative values for horizontal bars

Public Data Examples

Population by cityHospital beds by district

Common Mistakes

Too many categories make reading difficult; use horizontal bars for long labels

Accessibility:Good screen reader support; table alternative available
Alternative: Table for exact values

Line Charts

Trends and changes over time

Use when

  • •Time series
  • •Comparing multiple series
  • •Highlighting patterns and trends
  • •Continuous values

Avoid when

  • •Categorical data
  • •More than 5 series (cluttered)
  • •Few data points (use dot plot)
  • •Negative values without zero

Public Data Examples

Quarterly GDPMonthly inflation rate

Common Mistakes

Don't exclude zero on Y-axis when relevant; too many lines are hard to distinguish

Accessibility:Trend info in aria-describedby; keyboard support (arrow keys)
Alternative: Area chart for volume emphasis

Pie & Donut Charts

Proportions and composition of a whole

Use when

  • •Composition where sum = 100%
  • •3-7 categories
  • •Single data series
  • •Quick proportion insight

Avoid when

  • •More than 7 categories
  • •Comparing multiple series
  • •Similar values (hard to distinguish)
  • •Exact values (use table)

Public Data Examples

GDP by economic sectorStudents by education level

Common Mistakes

Too many segments; don't sort by size if illogical; avoid 3D effects

Accessibility:Limited support; MUST provide table alternative
Alternative: Bar chart for comparing proportions

Area Charts

Cumulative values and composition over time

Use when

  • •Volume over time
  • •Stacked composition
  • •Part-to-whole trends
  • •Magnitude emphasis

Avoid when

  • •Precise value reading
  • •More than 4 layers
  • •Overlapping series without transparency
  • •Negative values

Public Data Examples

Exports by sector (annual)Energy consumption by source

Common Mistakes

Layer order affects reading; largest category goes at bottom

Accessibility:Layers hard to distinguish; table alternative recommended
Alternative: Line chart for cleaner comparison

Scatter Plots

Correlation between two variables

Use when

  • •Exploring variable relationships
  • •Showing clusters and outliers
  • •Large number of points
  • •Correlation hypotheses

Avoid when

  • •Few data points (use dot plot)
  • •Time series
  • •Categorical data
  • •Without clear correlation

Public Data Examples

Healthcare spending vs life expectancyGDP per capita vs literacy rate

Common Mistakes

Don't assume causation; include trend line if relevant

Accessibility:Difficult for screen readers; table with values mandatory
Alternative: Bubble chart for third dimension

Geographic Maps

Geographic data distribution

Use when

  • •Spatial patterns
  • •Comparing regions
  • •Local variations
  • •Public sector and demographics

Avoid when

  • •Small geographic differences
  • •More than 7 colors
  • •Without context (add legend)
  • •Time series (use animation)

Public Data Examples

Population density by districtVaccination coverage by municipality

Common Mistakes

Avoid red-green palettes; use accessible palettes; add patterns for colorblind users

Accessibility:Provide table alternative; ARIA labels for regions; color contrast ≥3:1
Alternative: Bar chart for precise comparison

Tables

Precise values and accessibility

Use when

  • •Exact values
  • •Screen reader accessibility
  • •Preview for download
  • •Multiple measures per category

Avoid when

  • •Visual trends
  • •Large tables without pagination
  • •Too many columns
  • •Quick pattern insight

Public Data Examples

Statistical yearbookBudget execution by ministry

Common Mistakes

Missing column headers; no sorting; poor contrast; cluttered data

Accessibility:Best support; use scope attributes; headers for every column
Alternative: Column chart for visual display

Government Color Palette

Use consistent colors for recognizable government identity

Primary Blue#0D4077

Main elements, headers, primary actions

Secondary Blue#1a5290

Secondary elements, links

Interactive Blue#4B90F5

Hover states, focus, active elements

Accent Red#C6363C

Warnings, important data, accent

Color Accessibility

  • • All colors meet WCAG AA contrast (≥4.5:1 for text)
  • • Use Okabe-Ito palette for colorblind-safe charts
  • • Never rely on color alone - add patterns or labels
  • • Avoid red-green combinations
